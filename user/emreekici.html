Parts Implemented by Ömer Emre Ekici¶

STORE¶

  • The store page lists the games available on the website. Admins have the capability to add or delete games.

The store seen by a regular user

The store seen by an admin

Adding games to the store¶

  • Admins can add games to the store via the “Add game” link in the navigation bar.

The data about the game has to be provided.

The result page after the game is added.

The added game is now listed at the store.

Deleting games from the store¶

  • Games can also be deleted by admins.

The game to be deleted is checked

The selected game is deleted from the website

GAME PAGE¶

  • The game page can be reached by clicking the name of the game on the store. This page contains the data of the game and buttons for various features, such as purchasing, rating and (for admins) editing the game.

The game page

Rating a game¶

  • Users can rate the game.

Editing a game¶

  • Admins can change the information of the game

The new data has to be provided. If a field is left blank, the original information is preserved.

Purchasing a game¶

  • Users can purchase a game if they have enough balance. Admins have infinite balance.

PROFILE¶

In their profile, users can view their library of the games they own, the list of their friends and the friend requests they have sent and received.

Avatar¶

Users can upload an image as their profile photo by clicking the “Change avatar” button at the buttom of their profile page.

COMMUNITY¶

The community page displays the activities of the user and the user’s friends sorted by the date, descending. The blocked friends’ activities is not shown.
